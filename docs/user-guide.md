# AgriSupply User Guide (Current System)

## Who Uses AgriSupply
- Buyer: discovers products, places orders, pays, tracks deliveries, submits reviews.
- Farmer: lists products, manages incoming orders, updates fulfillment status.
- Admin: monitors users, products, orders, and analytics.

## Getting Started

### Register
1. Open app and select Register.
2. Choose role (buyer/farmer).
3. Enter details and submit.
4. Complete phone OTP if prompted.

### Login
1. Open Login screen.
2. Enter email/password.
3. App routes you to role dashboard after success.

If you forgot password:
- Use Forgot Password and follow reset flow.

---

## Buyer Guide

### 1. Browse and Search
- Open Buyer Home.
- Use categories and search.
- Open product detail for full information.

### 2. Cart and Checkout
- Add products to cart from product details.
- Review cart quantities and totals.
- Proceed to checkout and enter delivery details.

### 3. Payment
- Select payment method.
- Confirm payment.
- Wait for payment confirmation and order status update.

### 4. Tracking and Reviews
- View orders and tracking timeline.
- After delivery, submit product review.

---

## Farmer Guide

### 1. Product Management
- Create and edit product listings from farmer screens.
- Keep stock and pricing updated.

### 2. Order Processing
- Open incoming farmer orders.
- Confirm/reject, then progress through shipping/delivery states.
- Keep status accurate for buyer tracking.

### 3. AI and Analytics
- Use AI assistant features for farming support.
- Review analytics to monitor performance.

---

## Admin Guide

### 1. Dashboard and Oversight
- Open admin dashboard for platform summary.
- Monitor user, product, and order activity.

### 2. Moderation
- Review users/products.
- Apply verification, suspension, approval, and correction actions.

### 3. Reports and Settings
- Access analytics and exports.
- Adjust system-level settings.

---

## Notifications
Users receive notifications for important events such as:
- New order events
- Order status changes
- Payment status updates
- Administrative notices

Notification preferences can be managed in notification settings.

---

## Typical Status Lifecycle

### Order
pending -> confirmed -> processing/shipped -> delivered

### Payment
pending -> processing -> successful or failed

---

## Troubleshooting
- Login fails: verify credentials and account status.
- No products shown: check filters/search terms.
- Payment pending too long: verify callback status and retry if allowed.
- Missing notifications: check notification permissions and preferences.
